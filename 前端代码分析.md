# 法硕背诵小程序 - 前端代码分析与后端对接文档

## 一、项目概述

这是一个微信小程序项目，名为"法硕背诵"，主要面向法硕考试考生，提供知识卡片创建、学习、背诵等功能。

### 技术栈
- 微信小程序原生框架
- TDesign UI 组件库
- dayjs 日期处理库

### 项目结构

```
前端/
├── app.js                 # 应用入口
├── app.json               # 应用配置
├── app.wxss               # 全局样式
├── config/                # 配置文件
│   └── index.js           # 全局配置（useMock开关等）
├── utils/                 # 工具函数
│   ├── api.js             # API请求封装（核心）
│   ├── auth.js            # 认证相关工具
│   └── util.js            # 通用工具函数
├── services/              # 服务层（数据获取）
├── model/                 # 数据模型定义
├── pages/                 # 页面目录
│   ├── login/             # 登录页
│   ├── home/              # 首页（社区）
│   ├── create/            # 创作页
│   ├── profile/           # 个人中心
│   ├── library/           # 知识库详情
│   └── card/              # 卡片学习
├── components/            # 公共组件
├── custom-tab-bar/        # 自定义TabBar
└── style/                 # 样式文件
```

## 三、API 接口规范

### 基础配置

```javascript
const API_BASE_URL = 'http://192.168.43.95/api'; // 开发环境
// const API_BASE_URL = 'https://api.yourdomain.com/api'; // 生产环境
```

### 请求格式

所有请求默认使用 `application/json` 格式，自动携带 Token：

```javascript
header: {
  'Content-Type': 'application/json',
  'Authorization': 'Bearer <token>'
}
```

### 响应格式

```javascript
{
  "success": true,        // 业务是否成功
  "code": 0,              // 业务状态码
  "message": "操作成功",   // 提示信息
  "data": { ... }         // 返回数据
}
```

### 错误状态码处理

| HTTP状态码 | 说明 | 前端处理 |
|-----------|------|---------|
| 200-299 | 成功 | 正常处理 |
| 401 | Token过期 | 清除登录状态，跳转登录页 |
| 403 | 无权限 | 提示"无权访问" |
| 404 | 资源不存在 | 提示"资源不存在" |
| 429 | 请求频繁 | 提示"请求过于频繁" |

### 4.1 认证模块 (authAPI)

| 接口 | 方法 | 路径 | 说明 | 请求参数 |
|------|------|------|------|---------|
| 微信登录 | POST | `/auth/login` | 微信登录 | `{ code, userInfo }` |
| 获取用户信息 | GET | `/auth/me` | 获取当前用户 | - |
| 更新用户信息 | PUT | `/auth/profile` | 更新资料 | `{ nickname, avatar, bio }` |
| 刷新Token | POST | `/auth/refresh` | 刷新Token | - |

#### 登录接口详情

**请求：**
```javascript
POST /auth/login
{
  "code": "微信登录凭证",
  "userInfo": {
    "nickName": "用户昵称",
    "avatarUrl": "头像URL"
  }
}
```

**响应：**
```javascript
{
  "success": true,
  "data": {
    "token": "JWT Token",
    "userInfo": {
      "id": 1,
      "nickname": "用户昵称",
      "avatar": "头像URL",
      "bio": "个人简介"
    }
  }
}
```

## 八、后端开发建议

### 8.1 数据库设计建议

主要数据表：
1. `users` - 用户表
2. `libraries` - 知识库表
3. `chapters` - 章节表
4. `cards` - 卡片表
5. `study_records` - 学习记录表
6. `comments` - 评论表
7. `favorites` - 收藏表
8. `wrong_cards` - 错题表

### 8.2 认证流程

1. 前端调用 `wx.login()` 获取 `code`
2. 前端将 `code` 发送到后端 `/auth/login`
3. 后端使用 `code` 调用微信API获取 `openid`
4. 后端生成 JWT Token 返回给前端
5. 前端存储 Token，后续请求携带 Token

### 8.3 微信登录服务端API

```
GET https://api.weixin.qq.com/sns/jscode2session
  ?appid=APPID
  &secret=SECRET
  &js_code=CODE
  &grant_type=authorization_code
```

返回：
```json
{
  "openid": "用户唯一标识",
  "session_key": "会话密钥",
  "unionid": "用户在开放平台的唯一标识"
}
```

## 九、注意事项

1. **开发环境配置**：前端API地址使用局域网IP，后端需要配置CORS允许跨域
2. **Token过期处理**：前端已处理401状态码，后端需返回正确的HTTP状态码
3. **Mock模式**：前端 `config/index.js` 中 `useMock: true` 表示使用模拟数据
4. **分页参数**：统一使用 `page` 和 `pageSize` 参数
5. **时间格式**：建议使用 ISO 8601 格式或时间戳
